# SignalLoomOps

SignalLoomOps is a local-first InvoiceOps Copilot for marketing invoice processing. It ingests source documents, preserves source text and hash evidence, extracts invoice records, reviews exceptions, creates reusable deliverable cards, and optionally routes approved packets to workflow services.

## What it is for

Use SignalLoomOps to reduce manual hours spent on processing, reviewing, and paying marketing invoices:

1. Open an invoice, workbook, email export, PDF, DOCX, CSV, JSON, XML, HTML, Markdown, TXT, or log file.
2. Review extracted invoice fields, missing data, duplicates, thresholds, and approval route.
3. Ask local Qwen questions in the Copilot tab.
4. Open, revise, and save deliverable cards.
5. Route approved JSON to n8n, Make, Zapier, Slack, or a generic HTTPS webhook.

## First run from source

```powershell
python -m pip install -r requirements.txt
python run.py
```

Compatible launcher:

```powershell
python run_signalloom.py
```

## Local AI

SignalLoomOps uses Ollama + `qwen3:8b` by default. The app includes a one-click Local AI Setup / Repair screen. It can detect Ollama, start the service, pull the default model, and verify readiness. If local AI is offline, deterministic extraction and review still run.

## Build a Windows wizard installer

On a Windows build machine:

```powershell
.\build_win.ps1
```

Outputs:

```text
dist\SignalLoomOps\SignalLoomOps.exe
dist\installer\SignalLoomOps_Setup.exe
```

The setup wizard installs the app like a normal Windows application and includes the Python runtime inside the PyInstaller build. End users do not need to create a Python environment.

## Deliverables generated by the app

- AP Review Packet
- Slack Approval Message
- n8n / Make Payload
- Exception Checklist
- Qwen Draft cards from Copilot answers

Each card can be opened, revised with Qwen, and saved.

## Local storage

Data is stored under:

```text
~/.signalloom
```

This includes source/result history, service settings, and exported card content.

## Final stream build notes

- Qwen answers stream directly into the Copilot chat.
- Overview has a navigation shortcut; the actual document importer is on Ingest.
- The chat scrollbar is a slim modern custom control.
- Background helper commands are hidden to avoid flashing console windows.
- Use `build_win.ps1` to build the portable app and wizard installer.


### Chat pane clipping fix
This build removes the fixed wrap width that could cut off Qwen messages in narrower windows. The Copilot pane now prioritizes the chat area and resizes bubble wrapping dynamically.


## Chat polish build notes

- Copilot messages now render in a clean lab-style chat surface.
- Qwen responses use borderless assistant text, not boxed terminal-style bubbles.
- User prompts remain lightly wrapped on the right.
- Long answers and markdown-like tables are rendered in read-only wrapped text blocks to prevent clipping.
- Deliverable cards remain bordered because they are reusable outputs users can open, revise, and save.
- The loading overlay remains borderless, translucent, background-safe, and avoids fake percentage progress for unknown-duration tasks.

## InvoiceOps workflow focus

SignalLoomOps is organized around a practical invoice automation flow:

1. **Capture** invoices from files, workbooks, PDFs, email exports, CSV, JSON, and text while preserving source hash evidence.
2. **Match** invoice fields against PO, receipt, vendor, campaign, cost center, department, payment terms, and approver expectations.
3. **Approve** by detecting exceptions, duplicate risk, missing fields, threshold issues, and routing gaps.
4. **Pay / Handoff** by generating reviewed AP packets, Slack approval messages, exception checklists, and webhook-ready payloads for n8n, Make, Zapier, Slack, or generic HTTPS workflows.

The About tab includes the product purpose, Capture → Match → Approve → Pay playbook, audit-risk framing, creator attribution, DOI, ORCID, and Coffee License.
